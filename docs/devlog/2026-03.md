- Verified Dockerfile builds and WAL-G binary runs correctly on Alpine (gcompat for glibc compat)

### Decisions

- WAL-G cron inside postgres container (not sidecar) — one `apk add dcron`, avoids cross-container coordination, matches init-db.sh pattern

## 2026-03-03 — Testing Infrastructure Hardening

### Done

- Added diff-cover guardrails: 80% minimum coverage on changed lines in PRs, CI job runs `diff-cover` against `origin/main`, fails PR if new code drops below threshold
- Added coverage infrastructure to all 7 packages: `test:cov` scripts with lcov + JSON output, per-package thresholds (API: 33% lines/25% branches, Web: 50% lines), CI coverage report job aggregating all lcov files
- Enforced console.error/warn as test failures: global setup in both Vitest (`apps/api`) and Jest (`apps/web`) with `afterEach` assertions, allowlisted 38 intentional warnings across both test suites
- Fixed 12 console warnings across test suites (act warnings, mock setup issues, expected error logging)
- Added flakiness detection: `--sequence.shuffle` on API tests, retries disabled in CI, `.flaky.test.ts` suffix convention documented
- Added risk-based test matrix to `docs/testing.md`: minimum test layers per domain (7 domains), coverage by domain table, high-risk hotspot identification
- Codex plan review: addressed all 4 findings (type safety in setup files, suppressions doc, threshold justification, CI artifact naming)

### Decisions

- 80% diff-cover threshold: aggressive enough to prevent new low-coverage code, lenient enough to not block emergency fixes
- Per-package thresholds set at current coverage minus 5% buffer, ratcheted up monthly
- Console warning allowlist uses exact string matching, not regex — prevents silent drift
- Flaky test quarantine uses file suffix (`.flaky.test.ts`) rather than skip markers — visible in file listings and CI output
