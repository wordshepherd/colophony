- Updated `global-setup.ts` with upload infrastructure health checks (tusd + MinIO)
- Final results: 6/6 upload E2E, 20/20 submission E2E, 534+ unit tests, type-check clean

### Decisions

- Support both tusd v1 and v2 webhook payload formats for backward compatibility
- Use delayed invalidation (setTimeout) rather than websocket/SSE for webhook completion notification — simpler, sufficient for the use case

### Done

- Created `@colophony/api-contracts` workspace package with oRPC contract definitions for all 9 organization endpoints

- Implemented all 9 organization REST handlers calling existing service layer methods via `toServiceContext()`
- Scoped wildcard content-type parser inside Fastify plugin to avoid breaking tRPC/webhook body parsing
- Switched from contract-first `implement()` to server-first `os` procedure builders to resolve context type incompatibility
- Used `z.coerce.number()` in `restPaginationQuery` for REST query string param coercion
- Wrote 30 new tests (23 router + 7 error mapper), all 367 tests pass
- Addressed Codex branch review P1 findings:
  - Added `assertOrgIdMatch()` guard enforcing path `orgId` matches `X-Organization-Id` header context on all 6 org-scoped handlers
  - Created `memberMutationResponseSchema` matching actual DB row shape (no email join, includes `organizationId`/`updatedAt`) for `add` and `updateRole` contract outputs

### Decisions

- **oRPC over ts-rest**: ts-rest's Fastify adapter has no stable release for Fastify 5 / Zod 4 (only a 9-month-old RC with peer dep overrides). oRPC v1.13.5 provides stable Fastify 5 + Zod 4 support with zero overrides, contract-first design, built-in OpenAPI generation, and active maintenance.
- **Server-first approach**: oRPC's `implement(contract)` creates procedures with `Record<never, never>` initial context, incompatible with RestContext middleware. Using `os.$context<RestContext>()` procedure builders avoids the issue while contracts remain available for typed client generation.
- Contracts package (`@colophony/api-contracts`) is separate from the API app — will be consumed by future typed client package and OpenAPI generation.
- REST routes registered at `/v1/*` via `app.register(registerRestRoutes)` — inherits all Fastify hooks (auth, rate-limit, org-context, db-context, audit).
