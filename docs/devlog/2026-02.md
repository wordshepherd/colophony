## 2026-02-15 — Zitadel Webhook Hardening (Track 1)

### Done

- **PR #58** — Three defense-in-depth hardening measures for Zitadel webhook handler (13 files, +612/-60 lines)
- **Timestamp freshness check**: rejects events older than `WEBHOOK_TIMESTAMP_MAX_AGE_SECONDS` (default 300s) or >60s in the future; 400 response (Zitadel only retries 5xx)
- **Webhook-specific rate limiting**: per-IP Redis rate limiter via `preHandler` hook with Lua script, lazy connection, graceful degradation; fills the `/webhooks/` gap in global rate limiter
- **Out-of-order event guard**: `last_event_at` column on users table; atomic `setWhere` for upserts, compound `and(eq, or(isNull, lt))` for updates; stale-vs-not-found distinction via existence check
- Added `lastEventAt` column to users schema + manual migration (0006)
- Added `WEBHOOK_TIMESTAMP_MAX_AGE_SECONDS` and `WEBHOOK_RATE_LIMIT_MAX` env vars with Zod validation
- 13 new tests (26 total webhook tests), all 143 API tests pass
- Addressed Codex branch review (2 P2 findings, both fixed): gated audit logging on `result.rowCount` for update-only events; switched hardcoded `colophony:wh-rl:` Redis prefix to configurable `RATE_LIMIT_KEY_PREFIX`

### Decisions

- **Manual migration** — `drizzle-kit generate` TUI prompts can't be piped stdin in non-interactive shell; wrote `ALTER TABLE` directly
- **`setWhere` for upsert ordering** — Drizzle 0.44.7 supports this; makes ordering check atomic within the upsert (no extra query)
- **`lastEventAt` uses Zitadel's `creationDate`** not server `updatedAt` — different clocks, different semantics
- **Inline Lua script duplication** — not extracted to shared module; 2 call sites doesn't warrant the abstraction
- **Audit logging gated on mutation** — stale update-only events (rowCount=0) no longer produce audit entries

### Issues Found

- **Drizzle snapshot gap**: migrations 0004-0006 don't have corresponding `meta/*.snapshot.json` files; `drizzle-kit generate` interactive prompt about `audit_events_org_isolation_select` rename blocks non-interactive generation
- **Web type-check errors**: pre-existing failures in `@colophony/web` (PrismaClient import, missing tRPC routes) — unrelated to this work

- **tRPC Zod errors are 400, not 500**: tests and docs claimed `INTERNAL_SERVER_ERROR`; actual behavior is `BAD_REQUEST`. `ts-jest` never caught this because it transpiles without type-checking
- **Web tsc resolves API source via path alias**: `@prospector/api/*` → `../api/src/*` means web's `tsc --noEmit` type-checks NestJS decorators without `experimentalDecorators`
- **Prisma field name drift**: `previousStatus`/`newStatus` in code vs `fromStatus`/`toStatus` in schema; `userId_organizationId` vs `organizationId_userId` (Prisma generates names in schema declaration order)
- **`next-env.d.ts` lint error**: auto-generated triple-slash references trigger ESLint
- **`gcTime` vs `cacheTime`**: TanStack Query v4 uses `cacheTime`; `gcTime` is v5

### Decisions

- Pre-push (not pre-commit) for type-check/lint — keeps commits fast, catches errors before CI
- Pre-commit stays lean: secret scanning + Prettier only
- All these errors were invisible locally because `ts-jest` only transpiles (no type checking) and local Docker had full RLS setup
